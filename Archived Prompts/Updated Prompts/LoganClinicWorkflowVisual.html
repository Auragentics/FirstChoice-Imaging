📝 Medical Records Variables (records_* prefix):

  • {{records_caller_type}} — "patient", "parent_guardian", "ordering_provider", "non_ordering_provider", "attorney", "third_party"
  • {{records_release_required}} — "YES" or "NO" (auto-set based on caller_type)
  • {{records_patient_name}} — Patient's full name (= caller_name if patient)
  • {{records_request_type}} — "transfer_to_provider" or "radiology_report" (patient/guardian only)
  • {{records_patient_dob}} — Patient DOB (provider/attorney/3rd party only)
  • {{records_sms_link_sent}} — "YES" or "NO" — web form link texted?

📅 Scheduling Transfer Workflow (IMMEDIATE)

flowchart TD subgraph SCHEDULE["📅 SCHEDULING REQUEST"] A[/"Caller Requests Scheduling"/] --> B["Detect Scheduling Intent"] end subgraph IMMEDIATE["⚡ IMMEDIATE TRANSFER"] B --> C["Do NOT collect additional data"] C --> D["Trigger Transfer Tool Immediately"] end subgraph COMPLETE["✅ TRANSFER COMPLETE"] D --> E[/"Call Transferred to Scheduling Team"/] E --> F["Human Agent Handles Data Collection"] end

⚡ IMMEDIATE TRANSFER — No Data Collection:

  • When caller requests scheduling, immediately trigger the transfer tool
  • Do NOT collect patient name, DOB, or imaging service
  • The human scheduling team will gather all required information
  • Universal variables (caller_name, caller_phone) are already captured in Steps 1-2

✅ Closing Sequence

flowchart TD A["All workflows complete"] --> B["Pre-Close Data Completeness Check"] B --> C{"All variables present?"} C -->|"No"| D["Circle back — collect missing data"] D --> C C -->|"Yes"| E["Post-Collection Confirmation"] E --> F["SMS Consent Question"] F -->|"Yes"| G["sms_consent = YES"] F -->|"No"| H["sms_consent = NO"] G --> I["Close: 'Anything else?'"] H --> I I -->|"No"| J["'Have a wonderful day!'"] J --> K["NO REPEAT GOODBYES — End Call"]

✅ Pre-Close Data Completeness Check:

Agent silently verifies ALL required variables before closing:

  • Universal: caller_name, caller_phone, sms_consent
  • If medical_records: records_caller_type, records_patient_name, records_sms_link_sent, + conditional (records_request_type OR records_patient_dob)
  • If billing: billing_patient_name, billing_patient_dob, billing_reason
  • If lien: lien_patient_name, lien_patient_dob, lien_firm_name, lien_paralegal_name, lien_callback_phone, lien_clinic_location

📝 Post-Collection Confirmation Script:

"I've gathered all that information for our team. They will review it and follow up with you shortly."

📱 SMS Consent Script:

"To keep you updated on your request, would you like to receive status updates via text message to this phone number? Standard message and data rates may apply."

🧑‍💼 Live Operator Transfer (Last Resort)

⚠️ CRITICAL: Last Resort Only — 3-Step Gatekeeper Protocol

  • The Live Operator transfer is a LAST RESORT option
  • Agent must exhaust all other options using the 3-step protocol below
  • Use for: complaints, escalations, complex situations, non-imaging requests

3-Step Gatekeeper Protocol

flowchart TD A["Caller: 'I want to speak to a real person'"] --> B["Step 1 — DEFLECTION"] B --> B1["Agent: 'I can help you right here with scheduling, medical records, and billing. What specifically do you need?'"] B1 --> C{"Caller responds"} C -->|"States identifiable need"| D["Route to appropriate workflow"] C -->|"'I just want a human'"| E["Step 2 — INTERROGATION"] E --> E1["Agent: 'Our staff are currently likely with patients. If you tell me the reason, I can either handle it or ensure you get to the right person. Is this about an appointment or a bill?'"] E1 --> F{"Caller responds"} F -->|"Valid reason"| G["Step 3 — TRANSFER"] G --> G1["Transfer to Receptionist"] F -->|"Still vague/refuses"| H["Step 3 — END CALL"] H --> H1["'Cannot transfer without inquiry type. Please call back.'"]

When to Use

Scenario Action
Caller asks for "live person" at start of call Do NOT transfer — Step 1 Deflection
Caller has complaint or escalation Transfer to Receptionist after Step 2
Complex situation outside standard workflows Transfer to Receptionist after Step 2
Caller confused/frustrated after multiple attempts Transfer to Receptionist
Non-imaging request (vendor, job inquiry) Transfer to Receptionist after Step 2

📝 Transfer Phrase (After Exhausting Options):

"I want to make sure you get the help you need. Let me connect you with one of our staff members who can assist further. Just one moment."

🔄 Multi-Intent Call Example

Scenario: Caller needs medical records AND has a billing question

  1. Records Workflow: Agent identifies caller type (privacy screening), collects records_* variables, directs to web form
  2. Agent asks: "Anything else I can help with?"
  3. Caller mentions billing question
  4. Billing Workflow: Agent collects billing_* variables
  5. Agent asks: "Anything else I can help with?"
  6. Caller says no
  7. Pre-Close Check: Agent verifies all required variables are present
  8. Post-Collection Confirmation: "I've gathered all that information for our team..."
  9. SMS Consent: "Would you like to receive status updates via text?"
  10. Agent closes call

Final {{intents_handled}}: "medical_records,billing"

This allows downstream webhooks to route notifications based on which intents were handled.
