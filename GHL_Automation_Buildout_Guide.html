Workflow 2: Post-Transcript Actions TRANSCRIPT TRIGGER

Purpose: After the transcript is generated, routes SMS and email notifications based on intents_handled, records_caller_type, records_request_type, and sms_consent.

Total actions: ~25 (1 trigger + 1 gate + 3 intent branches with nested logic)

1
Trigger: Generate Transcript
Generate Transcript
SettingValue
Trigger NamePost-Transcript Actions

Why Generate Transcript?

This ensures the call transcript is available for staff emails. The trigger fires after transcript generation completes.

2
Gate Check: intents_handled is not empty
If/Else
SettingValue
Action NameGate: Has Intents?
ConditionContact > Intents Handled is not empty
IF intents_handled IS NOT EMPTY ├── YES → Continue to Step 3 (intent branches) └── NO → STOP (no further actions — scheduling/FAQ call)

Why This Gate?

Scheduling calls and general FAQ calls produce intents_handled = "". Without this gate, the workflow would evaluate all If/Else branches on every call and find nothing to do. The gate exits early.

Key Design Rules

1. Use CONTAINS for intents_handled

Never use EQUALS. Multi-intent calls produce "records,billing". Using "equals" would miss multi-intent matches.

2. Use EQUALS for all other fields

sms_consent, records_caller_type, and records_request_type are single-value enums — "equals" is correct.

3. All 3 intent branches are independent

They are NOT if/else-if/else. Each branch is its own If/Else at the same nesting level. A call with "records,billing" fires both Branch 1 and Branch 2.

4. Staff emails ALWAYS send

Emails are NOT gated by sms_consent. Only caller SMS is gated by consent.

5. SMS gate is per-branch

Each branch has its own sms_consent = YES check rather than one global check. This keeps logic clean if branches are added or removed later.

6. Every SMS includes "Reply STOP to opt out"

Required for A2P 10DLC compliance. No exceptions.

7. Form link placeholders

Replace [MEDICAL_RECORDS_FORM_LINK], [RADIOLOGY_REPORT_FORM_LINK], [PROVIDER_RECORDS_FORM_LINK], and [PROVIDER_RADIOLOGY_FORM_LINK] with actual First Choice Imaging URLs when available.
